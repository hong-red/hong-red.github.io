# 文明回响：让传统文化在AI时代焕发新生

## 项目概述

"文明回响"是一个融合AI技术的传统文化互动平台，旨在通过现代化的技术手段，让古老的传统文化以更生动、更有趣的方式走进人们的生活。平台分为儿童、青少年和老年三个板块，分别针对不同年龄段的用户需求，提供定制化的传统文化体验。

### 核心功能模块

#### 儿童板块：神兽诗故事
- **四大神兽主题**：青龙、白虎、朱雀、玄武，每个神兽都有独特的视觉风格
- **AI故事生成**：结合经典诗词与神兽传说，自动生成适合儿童的趣味故事
- **语音播放功能**：支持中文语音合成，让故事"活"起来
- **自动适配移动端**：无缝切换桌面端与移动端体验

#### 青少年板块：诗词与创作
- **诗词混创工坊**：AI协助续写诗词，支持多种情绪主题（孤独、漂泊、喜悦、思念、自由）
- **诗词信笺**：创意诗词展示，感受传统文化的美学魅力
- **书法展示**：浏览传统书法作品，提升审美修养

#### 老年板块：时空对话
- **与苏轼对话**：通过AI技术，实现与古代文豪的跨时空交流
- **简洁易用的界面**：专为老年用户设计，操作简单直观

## 技术实现

### 技术栈
- **前端**：HTML5 + CSS3 + JavaScript (ES6+)
- **后端**：Node.js + Express
- **AI服务**：Moonshot AI API
- **部署方案**：GitHub Pages + Vercel

### 架构设计
1. **前端**：静态文件部署在GitHub Pages，通过GitHub Actions实现自动部署
2. **后端**：Express服务部署在Vercel，提供AI API接口
3. **AI集成**：通过Moonshot AI API实现故事生成、对话和诗词续写功能

### 核心API端点
- `POST /api/kids/story`：生成儿童神兽诗故事
- `POST /api/sushi`：与苏轼进行时空对话
- `POST /api/creation/continue`：AI协助续写诗词

## 开发过程中的问题与解决方案

### 1. 电脑端故事播放功能失效
**问题描述**：用户反映电脑端无法播放故事，点击播放按钮没有反应。

**排查过程**：
- 检查代码发现，播放和暂停按钮被意外移除
- 语音合成对象`utterance`未正确初始化
- 缺少浏览器兼容性检查

**解决方案**：
- 恢复播放和暂停按钮，重新实现`playStory()`和`pauseStory()`函数
- 添加`utterance`变量声明和初始化
- 增强播放逻辑，添加浏览器兼容性检查和错误处理
- 添加详细的调试日志，便于问题定位

### 2. 跨域请求问题
**问题描述**：前端从GitHub Pages访问Vercel后端时，出现跨域请求错误。

**解决方案**：
- 在Express后端添加CORS中间件，允许所有域访问
- 配置Vercel的跨域策略，确保API可以被外部调用

### 3. AI生成速度慢
**问题描述**：AI生成故事和对话的速度较慢，用户体验不佳。

**解决方案**：
- 优化前端加载状态，添加友好的加载提示
- 设置合理的超时时间（30秒），避免用户长时间等待
- 实现错误重试机制，提高成功率

最后还是用了梯子才行

### 4. 移动端适配问题
**问题描述**：部分功能在移动端显示异常，交互不流畅。

**解决方案**：
- 为每个页面创建专门的移动端版本（如`m-kid.html`）
- 使用CSS媒体查询，优化不同屏幕尺寸的显示效果
- 简化移动端交互，确保操作便捷

# 开发笔记：反复遇到的问题与解决方案

在开发"文明回响"项目的过程中，我遇到了一些反复出现的问题，通过不断探索和调试，最终找到了解决方案。以下是这些问题的整理和记录：

## 1. 语音合成功能不稳定

**问题现象**：
- 有时能正常播放，有时无法播放
- 不同浏览器表现不一致
- 移动端和桌面端体验差异大

**解决方案**：
```javascript
function playStory() {
  if (!utterance || !currentStory) {
    console.error('语音合成对象或故事内容不存在');
    return;
  }

  // 检查语音合成是否可用
  if ('speechSynthesis' in window) {
    try {
      window.speechSynthesis.cancel();
      window.speechSynthesis.speak(utterance);
      console.log('开始播放语音');
    } catch (error) {
      console.error('播放语音时出错:', error);
      alert('播放失败，请检查浏览器语音设置');
    }
  } else {
    console.error('当前浏览器不支持语音合成 API');
    alert('当前浏览器不支持语音合成功能');
  }
}
```

**关键改进**：
- 添加浏览器兼容性检查
- 完善错误处理机制
- 增加调试日志
- 提供友好的用户提示

## 2. API调用失败处理

**问题现象**：
- API调用超时，导致页面长时间无响应
- 错误信息不明确，用户不知道发生了什么
- 缺少重试机制

**解决方案**：
```javascript
// 自动超时处理
const controller = new AbortController();
const timeoutId = setTimeout(() => controller.abort(), 30000); // 30秒超时

try {
  const response = await fetch(`${API_BASE_URL}/api/kids/story`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      beast: beast,
      poemTitle: title,
      poet: poet,
      poemText: text
    }),
    signal: controller.signal
  });

  clearTimeout(timeoutId);

  if (!response.ok) throw new Error("API 请求失败: " + response.status);

  // 处理成功响应
} catch (err) {
  clearTimeout(timeoutId);
  console.error(err);
  // 显示友好的错误提示
}
```

**关键改进**：
- 添加30秒超时控制
- 显示具体的错误状态码
- 提供重试按钮
- 优化错误提示的用户体验
